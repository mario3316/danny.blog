## 구현기능

**게시글 목록 페이지**

-   Next.js의 App router의 동적 세그먼트로 페이지 디렉토리 구현, 라우트를 빌드 타임에 생성
-   generatetaticParams()로 동적인 경로를 받아와 게시글 카테고리 분류

**게시글 상세 페이지**

-   게시글을 사전에 서버에서 렌더링되도록 구성
-   `react-mdx-remote`를 활용 사전에 작성한 mdx 파일을 HTML로 파싱 후 렌더링

**목차 사이드 바(ToC)**

-   Intersection Observer API를 사용하여 DOM 요소 관찰
-   특정 DOM 요소(포스트의 h2, h3 태그)로 이동할 수 있는 ToC 사이드바 구현
-   로딩 시 trasition 애니메이션 구현

**검색엔진 최적화(SEO)**

-   generateMetadata()로 동적 metadata 생성
-   Static Site Generation으로 정적인 페이지 구성
-   시맨틱 태그를 사용하여 페이지 구조 표현
-   Image 태그의 alt 속성 활용

**그 외**

-   댓글 기능
-   다크모드 / 라이트모드
-   글 목록 리스트 그리드형 / 리스트형 분리
-   반응형 레이아웃 (sm, md)

## 기술스택

### Next.js

Next.js의 Static Site Generation 기능으로 정적인 페이지가 많은 블로그 특성상 보다 빠른 렌더링이 가능합니다.
또한 Next.js는 React 기반의 프레임워크로 Image, Video, Font 등 다양한 방면에서의 최적화가 가능하다는 점도 선정 이유로 작용했습니다.

### Typescript

Typescript를 채용함으로써 컴파일 과정에서 타입 오류를 미리 찾아낼 수 있다는 점이 개발 과정에서 매우 중요하게 작용했습니다.
특히나 블로그에 올릴 포스트 등에 들어갈 여러 데이터의 타입을 미리 정의해놓고 개발을 진행하니 중간에 빼먹은 데이터나 형식이 맞지 않는 데이터를 입력했을 때 문제점을 찾고 디버깅을 하기가 훨씬 수월해졌습니다.

### Tailwindcss

Tailwindcss를 사용해서 보다 일관된 디자인을 구성할 수 있었고, CSS 파일을 따로 분리하지 않고 className에 스타일을 적을 수 있게 됨으로써 코드의 길이를 줄일 수 있게 되었습니다. 또한 반응형 디자인이나 간단한 애니메이션을 구현하는 것도 쉬워졌습니다.
