## 구현기능
- 포스트 목록 페이지(그리드형 / 리스트형)
- 포스트 상세 페이지
    - ToC 사이드바
    - 댓글 기능
- 프로젝트 설명 페이지
- 다크모드

## 기술스택 선정 이유

### Next.js
Next.js의 Static Site Generation 기능으로 정적인 페이지가 많은 블로그 특성상 보다 빠른 렌더링이 가능합니다.
또한 Next.js는 React 기반의 프레임워크로 Image, Video, Font 등 다양한 방면에서의 최적화가 가능하다는 점도 선정 이유로 작용했습니다.

### Typescript
Typescript를 채용함으로써 컴파일 과정에서 타입 오류를 미리 찾아낼 수 있다는 점이 개발 과정에서 매우 중요하게 작용했습니다.
특히나 블로그에 올릴 포스트 등에 들어갈 여러 데이터의 타입을 미리 정의해놓고 개발을 진행하니 중간에 빼먹은 데이터나 형식이 맞지 않는 데이터를 입력했을 때 문제점을 찾고 디버깅을 하기가 훨씬 수월해졌습니다.
