## 구현기능

**API 및 Axios 모듈화 작업**

-   Token이 필요한 Axios request와 아닌 request를 구분하여 구현
-   Axios interceptor를 사용하여 권한 만료로 요청 실패 시 요청을 가로채어 새 refreshToken을 발급받는 기능 구현
-   길어져서 읽기 힘든 API 요청 경로를 메소드화한 API 모듈을 구현하여 코드의 가독성과 협업의 편의성을 늘림
-   목업 데이터을 사전에 제작하여 프론트엔드 화면 개발속도 상승

**컴포넌트 개발**

-   반복되는 코드 및 길어지는 코드를 컴포넌트화 하여 개발 효율 증가
-   useRef와 document.addEventListener를 사용, 모달의 클릭 이벤트를 감지해 클릭한 타겟 DOM 요소와 모달 내부 DOM 요소를 비교하여 다른 경우 모달이 꺼지도록 하는 커스텀 훅을 구현

**동화책 관련 기능**

-   동화책 뷰어
-   동화 챌린지
-   동화 상세정보
-   메인화면

**회원관리 기능 전반 개발**

-   FormData API를 활용하여 Image 파일을 업로드하여 프로필 사진으로 변경하는 기능 구현
-   Zustand를 사용하여 회원 정보를 전역 상태로 관리, 회원 정보를 필요로 하는 여러 컴포넌트 간의 데이터 전달 개선
-   회원정보를 필요로 하는 컴포넌트들이 전역 상태를 관리하는 store를 구독하는 Observer 패턴으로 구현
-   Localstorage에 민감하지 않은 회원 정보만 저장, 브라우저가 꺼져도 일정 시간 동안은 로그인이 유지되도록 하여 사용자 경험 개선

## 기술스택

-   보다 일관된 디자인 개발을 위해, `Tailwindcss`를 도입하였습니다. 또한, `Styled-components`와 결합하여 Css-in-JS 코드를 작성해 개발 효율성을 높이도록 했습니다.
-   `Redux` 보다 보일러 플레이트 코드가 적고, 사용성이 간편한 `Zustand` 를 도입해 전역 상태를 관리했습니다.

## 프로젝트 리뷰

-   개발 팀장을 맡으면서 일정 및 역할 배분을 통해 리스크를 관리하는 방법을 고민하고 배울 수 있었습니다.
-   좋은 UI에 관한 글이나 UX를 고려한 디자인 등에 관한 글을 많이 찾아보았습니다. 개발자 칼럼 플랫폼, 기술 블로그를 구독하고 참고하고 있습니다.
-   이미지 뷰어를 만들때 react-slick, @ridi/react-viewer 등 다양한 레퍼런스와 라이브러리를 참고해 책장이 넘겨지는 이미지 뷰어를 구현했습니다.
