## 4. 컨벤션 정하기

혼자 하는 일이지만, git을 사용할 때 꾸준히 컨벤션을 사용하는 습관을 들여놔야 할 것 같다. 나중에 내 코드나 커밋 기록을 돌려볼 때도 도움이 되고

```jsx
Feat: 새로운 기능 추가
Refactor: 코드 리팩토링
Fix: 버그 수정
Env: 개발 환경설정
Style: 코드 스타일 수정
Design: CSS 등 디자인 추가 / 수정
Comment: 주석 추가 / 수정
Docs: 내부 문서 추가 / 수정
Test: 테스트 추가 / 수정
Chore: 빌드 관련 코드 수정
Rename: 파일 및 폴더명 수정
Remove: 파일 삭제
```

### 디렉토리 구조 설계

```jsx
public/
	assets/
		posts: 게시글에 사용할 이미지 등 저장

src/
	app
	lib: hook
	components: 여러 페이지에서 공용되는 컴포넌트
	config: const, type, style
	layouts: 전체적인 화면 레이아웃(header, footer, sidebar 등)
	posts: 게시글 mdx 파일 저장

```

```jsx
// 일반적인 게시글
src / posts / [tabs] / [categories] / [slug] / page.tsx;

//게시글 원본 위치
posts / [categories] / [title] / content.mdx;
```
